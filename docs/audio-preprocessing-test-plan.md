# 오디오 전처리 테스트 계획

이 문서는 회의 음성 전처리 전략을 결정하기 위한 테스트 계획이다.

기준일:

- 2026-04-29

목표:

1. 전처리 모드가 Cohere STT 정확도에 실제로 도움이 되는지 확인
2. 화자 분리 및 타임라인 정렬에 부작용이 없는지 확인
3. 기본값으로 둘 전처리 모드를 결정
4. 사용자 설정 UI 문구와 기본값을 근거 있게 정리

## 1. 비교 대상 모드

1차 비교 대상:

- `off`
- `auto`
- `loudnorm`
- `speechnorm` 후보

보류:

- `dynaudnorm`
  - 현재 짧은 저음량 샘플에서는 효과가 확인되지 않았다.
  - 자동 기본값이 아니라 실험 모드로만 유지한다.

- `silenceremove`
  - diarization / 타임스탬프 정렬에 미치는 영향이 커서 1차 비교 대상에서 제외한다.

- `denoise`
  - 한국어 자음/말끝 손실 위험이 있어 별도 실험으로 분리한다.

## 2. 테스트셋 구성

최소 1차 테스트셋:

- 총 6~10개 파일

권장 2차 테스트셋:

- 총 15~20개 파일

파일 유형:

1. 정상 음량, 조용한 환경
2. 작은 음량
3. 큰 음량
4. 화자 간 음량 차이가 큰 회의
5. 배경 소음 포함
   - 에어컨
   - 키보드
   - 마우스 클릭
   - 책상 충격음
6. 잔향/회의실 울림
7. 휴대폰 녹음
8. 긴 회의 파일
9. 한국어 고유명사/사내 용어 다수 포함
10. 영어/약어 혼용 발화 포함

샘플 길이 권장:

- 짧은 샘플: 30초 ~ 2분
- 중간 샘플: 3분 ~ 10분
- 긴 샘플: 10분 이상

## 3. 샘플별 메타데이터

각 테스트 파일에는 아래 메타데이터를 같이 기록한다.

- 파일명
- 길이
- 녹음 장치
- 녹음 장소
- 화자 수
- 예상 문제
  - 작은 음량
  - 잡음
  - 울림
  - 영어 혼용
  - 고유명사
- 원문 정답 유무
  - 있으면 참조 transcript 연결

## 4. 평가 항목

각 전처리 모드마다 아래를 비교한다.

1. STT 정확도
   - 한국어 고유명사 오인식
   - 영어/약어 오인식
   - 작은 목소리 누락 여부
   - 문장 경계 자연스러움

2. 화자 분리 영향
   - 화자 구간 경계 악화 여부
   - speaker alignment 어긋남 여부

3. 결과물 영향
   - 최종 summary 품질
   - action item / 결정사항 추출 품질

4. 처리 비용
   - 전처리 시간 증가량
   - 전체 분석 시간 증가량

## 5. 평가 방식

정량 평가:

- 가능하면 샘플별 참조 transcript 준비
- WER/CER 또는 간단한 편집거리 비교

정성 평가:

- 고유명사
- 숫자
- 날짜
- 담당자명
- 영어 혼용 표현

권장 표:

```text
샘플 | 모드 | STT 품질 | 고유명사 | 작은 음량 | 화자정렬 | 요약 품질 | 처리시간 | 비고
```

## 6. 통과 기준

기본값 후보가 되려면 아래를 만족해야 한다.

1. 전체 샘플에서 `off` 대비 의미 있는 품질 향상 또는 최소 동등
2. 화자 분리 품질 저하가 없어야 함
3. 긴 파일에서 처리시간 증가가 과도하지 않아야 함
4. 특정 샘플에서만 좋아지고 전체적으로 불안정하면 기본값으로 채택하지 않음

## 7. 실행 순서

1. 현재 로컬 샘플 정리
2. 부족한 유형 샘플 수집
3. `off / auto / loudnorm` 1차 비교
4. 필요 시 `speechnorm` 추가
5. 승자 모드를 기본값 후보로 지정
6. 그 결과를 UI 설정 문구와 도움말에 반영

## 9. 현재 판단

2026-04-29 기준:

- `off`와 `loudnorm`은 실제로 다른 출력 파일을 만든다.
- 현재 짧은 저음량 샘플에서는 `loudnorm` 효과가 확인됐다.
- 현재 짧은 저음량 샘플에서는 `dynaudnorm` 효과를 확인하지 못했다.
- 따라서 다음 비교의 중심은 `off / auto / loudnorm`이다.
