# Local Meeting AI - Roadmap

본 문서는 Local Meeting AI 데스크탑 앱의 향후 발전 방향과 추가 기능 개발 계획을 담고 있습니다.

## 🎯 단기 목표 (Phase 1)
- [x] 오프라인 구동용 핵심 파이프라인(CLI) 개발 완료 (ffmpeg, STT, Diarization, LLM Summary)
- [x] 모델 로컬 보관 및 교체 자동화 (`download_models.py`, `config.json` 매핑)
- [x] 백엔드 예외 처리 강화 (OOM 방어, 타임아웃 방어)
- [x] 데스크탑 API 사이드카 (FastAPI) 연동
- [x] 데스크탑 프론트엔드 (React + Vite) 디자인 시스템 및 UI 구축
- [ ] 파이썬 API와 React 프론트엔드 간의 실시간 진행률(Progress) 연동 로직
- [ ] Tauri를 이용한 최종 데스크탑 `.exe` 패키징

## 🚀 중장기 목표 (Phase 2 & Phase 3)

### 1. RAG 기반 회의록 대화/검색 기능 (채팅 UI)
- **개요:** 생성된 회의록(TXT, MD)들을 로컬 벡터 데이터베이스(예: ChromaDB, FAISS)에 임베딩하여 저장.
- **기능:**
  - "지난번 A프로젝트 예산 회의 때 B대리님이 뭐라고 하셨지?" 와 같이 자연어로 과거 회의 내용을 검색.
  - 현재 요약된 회의 내용에 대해 추가로 "이 회의에서 결정된 다음 스텝을 표로 다시 정리해 줘" 등의 후속 대화 가능.
- **기술 스택 후보:** `langchain`, `chromadb`, 로컬 임베딩 모델(예: `bge-m3` 등 다국어 지원 모델).

### 2. 다중 파일 및 일괄 처리(Batch Processing)
- 여러 개의 녹음 파일을 한 번에 업로드하고 백그라운드에서 순차적으로 텍스트 변환 및 요약을 수행.

### 3. 사용자 커스텀 사전 (고유명사 패치)
- 회사 내부의 고유명사, 약어, 인명 등을 사전에 등록하여 STT 과정에서 우선적으로 인식되도록 보정하는 기능 도입.

### 4. UI/UX 고도화
- 파형(Waveform) 표시 및 오디오 플레이어 내장 (화자별 구간 색상 표시).
- 잘못 인식된 STT 텍스트나 화자 정보를 클릭 한 번으로 수정할 수 있는 에디터 모드.
